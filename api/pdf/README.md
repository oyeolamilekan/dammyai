# `api/pdf` standalone PDF worker

This directory contains the standalone HTML-to-PDF service for DammyAI.

It is implemented as a Hono app running on Cloudflare Workers and uses Cloudflare Browser Rendering with `@cloudflare/puppeteer` so PDFs are generated by a real browser instead of a lightweight PDF library.

## Purpose

The worker exists to keep PDF generation outside the Convex runtime while still supporting:

- good HTML/CSS rendering fidelity
- browser-based print layout
- reusable PDF generation for research reports and other future exports

Convex calls this service through `convex/lib/pdfApi.ts`.

## File layout

```text
api/pdf/
├── src/app.ts         # Hono routes and error handling
├── src/index.ts       # Worker entrypoint and shared exports
├── src/render.ts      # Browser-based HTML -> PDF rendering
├── src/request.ts     # Request parsing and filename normalization
├── src/types.ts       # Shared worker types and constants
├── wrangler.jsonc     # Cloudflare Worker configuration
└── README.md          # This file
```

## Runtime flow

1. `src/app.ts` receives the request.
2. `src/request.ts` validates JSON or raw HTML input.
3. `src/render.ts` launches the Cloudflare browser binding and renders the HTML.
4. The worker returns a PDF response with the correct content headers.

## API routes

### `GET /`

Returns basic service metadata.

### `GET /health`

Returns a simple health response:

```json
{ "ok": true }
```

### `POST /pdf`

Accepts HTML and returns a PDF.

Supported request formats:

#### JSON body

```json
{
  "html": "<h1>Hello</h1><p>World</p>",
  "title": "Hello",
  "fileName": "hello.pdf"
}
```

#### Raw HTML body

Send raw HTML as the body and optionally provide:

- `title` query param
- `fileName` query param

## Response

Successful responses return:

- `Content-Type: application/pdf`
- `Content-Disposition: attachment; filename="..."`

Validation failures return JSON with status `400`.

Unhandled rendering failures return JSON with status `500`.

## Local development

Run the worker locally with:

```bash
bun run dev:pdf-api
```

This uses the `wrangler` config at `api/pdf/wrangler.jsonc`.

The browser binding is configured with `remote: true`, so local development uses Cloudflare's remote browser runtime instead of a local browser emulator.

## Deployment

Deploy with:

```bash
bun run deploy:pdf-api
```

## Wrangler config

Current `wrangler.jsonc` config includes:

- worker name: `dammyai-pdf-api`
- entrypoint: `src/index.ts`
- `nodejs_compat`
- browser binding: `BROWSER`
- `remote: true` for local development

## Rendering behavior

The worker renders PDFs with:

- A4 page format
- print backgrounds enabled
- CSS page size support
- browser-driven layout through Puppeteer

The HTML sent from Convex is already wrapped with print-oriented CSS and pagination rules before it reaches this worker.

## Convex integration

Convex uses this worker through the `PDF_API_BASE_URL` environment variable.

Example:

```bash
PDF_API_BASE_URL=https://your-worker.your-subdomain.workers.dev
```

When research delivery runs:

1. Convex generates the research report HTML.
2. Convex wraps the report with print-friendly PDF styles.
3. Convex sends that HTML to this worker.
4. The worker returns a PDF.
5. Convex uploads the PDF to Telegram.
